Retail Churn Case Study

1. Problem Statement & Business Goal

A global retail brand is facing a 15% increase in customer attrition. The goal is to use the CRISP-DM framework to identify high-value customers likely to churn and implement a proactive retention strategy using Knowledge Intelligence.

Phase 1 & 2: Understanding

Data Inputs: Historical CRM data, Transaction logs (SQL), and Customer Support tickets.

Actions: Defined churn as "No purchase in 90 days." Used Exploratory Data Analysis (EDA) to find correlations.

Phase 3: Data Preparation

Processing Steps: Handled 5% missing values via median imputation. Integrated Python libraries (Pandas).

Feature Engineering: Created "Recency, Frequency, Monetary (RFM)" scores.

Output: Finalized Training Set (CSV/Parquet)

Phase 4: Modeling

Tools: Scikit-Learn using Random Forest and XGBoost algorithms.

Technique: Hyperparameter tuning via Grid Search to optimize the F1-score.

Output: Optimized Predictive Model (Pickle file)

Phase 5: Evaluation

Metrics: Achieved 88% Accuracy and 82% Recall. Verified model alignment with business goals.

Knowledge Intelligence: Expert review confirmed "at-risk" features match market trends.

Output: Model Performance Evaluation Report
